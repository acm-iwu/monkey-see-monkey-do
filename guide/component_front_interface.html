Front-End: Interface & Design

This component deals with creating the layout, design, and form controls.

Requirements

You will need to create an interface that has the following form controls:

  • A way to select the monkey's sex.
  • A way to set the value of the "left ear". This is essentially a three digit binary number (e.g., 000, 010, 100, 110).
  • A way to set the value of the "right ear". Possible values are the same as the left ear.
  • A way to select the group. There are several types of groups, so a dropdown would probably be best for this. These group values depend on the database, so they will be filled via JavaScript.

Make sure to keep in mind when selecting what kind of form controls you use that speed is of utmost importance to the users. For example, a dropdown for the monkey's sex would not be as ideal as two buttons. Also, for each form control, you'll want the user to be able to toggle whether it is in use, because that information might not be known. This means that you'll need to use things like toggle buttons.

You need to create a table that displays the matching monkeys. You should display the following fields: ID, Sex, Birth Year, Earnotches*, Group, Notes. Note that earnotches is just the left ear value concatenated by a hyphen and the right ear value.

Finally, there are two other things that will be useful to the user. Since the database can only be updated with Internet access, it would be useful to display the last time it was updated on the screen. This would be filled in with JavaScript. Additionally, it would be useful to have a button somewhere that resets all of the form parameters so that they can easily identify multiple monkeys quickly.

Although the researcher specifically said an iPad would be used, try to make the site compatible with an iPad as well as an iPhone sized screen. When using Bootstrap (see Tools below) to create the layout, make sure to follow its grid system which will make the site responsive and adapt to an iPhone as well as an iPad. If this sounds confusing now, just keep it in mind, and it should make sense when you are actually working with Bootstrap.

Coordination with Other Components

You will need to call a function when any form field is updated. You will coordinate this with the Front-End Database JavaScript team. You may also want to create a link to the page where you can update the database created by the Database Upload Form team.

Tools

  • Bootstrap: Bootstrap is a CSS framework that is commonly used to create web applications. It includes many CSS classes and JavaScript components that are used in web applications. It also uses responsive design, so the design that you construct will adapt to both an iPhone and an iPad (as well as a desktop browser if you wanted).
  • jQuery: jQuery is a required component of Bootstrap, and you will use this JavaScript library to make some of the interactive components of the form work.

Steps

  1. Get familiar with Bootstrap. Bootstrap is a CSS framework commonly used for web application interfaces. CSS is a language that is used to define the style of a web page. Review the examples provided on the Bootstrap website, and copy the source of one into an HTML file called index.html. Then, you need to download Bootstrap and extract the contents inside the main directory. Finally, make sure that all of the links to CSS and JavaScript files point to the files you just downloaded. You may also need to copy a CSS file that it links to, and you can see what it links to by viewing the source.
  2. Add the necessary form components. Copy Bootstrap's form examples to create all of the form inputs as described in the requirements. To conserve space, it might be a good idea to use the horizontal form layout.
    • Make sure that you read the requirements closely and use your best judgement to pick form elements. You can get an idea of what kind of form elements are available by reading through the form examples or by going to the W3Schools page on forms.
    • Keep in mind that you can use buttons in Bootstrap. Another thing that might be useful are button groups, which could be used for something like the monkey's gender.
    • Something that Bootstrap does not include but that could be useful are toggle buttons. I have created an example of what I mean. The Front-End JavaScript team will make the buttons toggle-able, so all you need to worry about is creating the button elements themselves.
  3. Create the table. Review Bootstrap's table examples and create a table with the fields mentioned above. It would probably look best if this were positioned on the bottom of the screen.
  4. Add other components. The requirements also call for a spot to show when the database was last updated. Just leave a div element there with an ID that will let the Front-End: Database JavaScript team work on. Secondly, you need to create a button somewhere that clears out all of the form elements. The aforementioned team will also make this work, so you just need to create the button.
