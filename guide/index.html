Overview

A web app to filter and identify monkeys in the field based on physical cues.

Background

The Psychology department occasionally goes to Puerto Rico to perform studies on monkeys. It is important that they be able to identify these monkeys while on the field. Each monkey has a unique identifier, which is actually planted on the monkey, but it is often times impossible to read for various reasons.

The owners of the monkeys keep a database of the monkeys which includes each monkey's ID and some physical characteristics of the monkey. It is possible for our researchers to easily identify these physical characteristics of the monkeys when observing them. Therefore, the researchers would like some sort of program that lets them plug in these physical traits and get a list of monkeys matching them.

Constraints

  • They would like to use iPads.
  • They do not have Internet access while on the field (though they do where they are spending their nights).

Implementation Overview

There are many ways to go about this. We could create a native iPad app or create a web app. A native iPad app would require us to pay money for the iOS developer fee, and is more difficult to get started with, so we decided to go with the web app. Note that it is possible to create a web app that gets cached on an iOS device so that Internet is not required.

Front-End

For the front end, we are going to have a bunch of form controls that let you filter the monkeys. As the monkeys get filtered out, we will update a table showing matching monkeys in real time.

Back-End

The database gets updated approximately once a year. We need an interface that allows the Psychology department to update the database when it comes around. We also need to deliver this to the front-end when we load the site.

Constraints

Since we will want to put this on IWU's Sun server once completed, it would be easiest to use PHP for the back-end scripting language.
